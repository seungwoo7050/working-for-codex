현재 레포지토리는 4개의 프로젝트를 포함하고있습니다.

각각의 프로젝트는 독립적으로 빌드 및 실행될 수 있는 완전히 분리된 프로젝트입니다.

1. `spring-patterns/` - Spring 기반의 백엔드 서비스 프로젝트
2. `native-video-editor/` - 네이티브 기반의 비디오 편집기 프로젝트
3. `modern-irc` - C++17 기반의 IRC 서버 프로젝트
4. `ray-tracer` - C++17 기반의 레이 트레이싱 렌더러 프로젝트

각 프로젝트의 최종 버전정보는 CLONE-GUIDE.md 파일에 명시되어 있습니다.

각 프로젝트의 테스트가 부족한지 여부를 점검하고, 부족한 부분이 있다면 추가 테스트 코드를 작성하시기 바랍니다.

이후 프로젝트의 유닛/빌드/e2e 테스트를 시행 후 정상 동작하는지 확인하시기 바랍니다.

추가된 테스트를 포함하여 모든 테스트가 정상적으로 통과하지 않는다면 테스트가 잘못된 것인지, 소스코드가 잘못된 것인지 면밀히 검토후 수정과 테스트를 반복하시기 바랍니다.

모든 테스트를 통과한 후 CI/CD 파이프라인을 위해 루트위치에 .github/workflows/ 를 업데이트하세요.

각 프로젝트의 README.md 파일을 검토하여 최신 상태인지 확인하고, 필요한 경우 업데이트하시기 바랍니다.

모든 변경사항을 커밋하고 푸시한 후, 각 프로젝트의 주요 변경사항과 추가된 테스트에 대한 간략한 설명을 포함한 PR을 생성하세요.

마지막으로 4개의 프로젝트에 대한 포트폴리오로서의 가치평가문서를 루트위치에 작성하세요. 이 문서에는 각 프로젝트의 주요 기능, 사용된 기술 스택, 테스트 커버리지, CI/CD 파이프라인 구성 등에 대한 설명이 포함되어야 합니다.
